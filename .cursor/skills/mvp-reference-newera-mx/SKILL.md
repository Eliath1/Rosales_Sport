# MVP Reference (New Era MX)

## Instructions

1. Treat New Era MX as a reference - not a clone; document intentional deviations.
2. Wave 0 MVP: quote intake, email CRM, basic catalog browse - no full e-commerce checkout unless spec says otherwise.
3. Compare each proposed feature against reference doc sections (navigation, product detail, customization).
4. Prefer proven patterns over novelty for first release in Mexico market.
5. Capture gaps as future-wave items in feature specs.

## Key Workflows

### MVP scope check

```
- [ ] Feature listed in mvp-newera-mx-reference?
- [ ] Required for Wave 0 or later wave?
- [ ] Build vs buy decision documented (see build-vs-buy-mvap)
- [ ] i18n and MX payments impact assessed
```

### Reference-driven UX review

- Homepage and category navigation
- Product detail (variants, imagery)
- Customization / team order entry points
- Contact or quote CTA placement
